Large vision-language models (VLMs) have made remarkable strides on visual benchmarks, yet they
still exhibit systematic, repeatable failure modes on tasks that require structured visual reasoning.
A chart that demands multi-step data extraction, a high-resolution image that requires isolating a
sub-region, a mathematical diagram that calls for compositional interpretation: these cases expose
not random errors but consistent gaps in strategy and perception.
Current approaches to closing these gaps fall into two camps, both expensive.
Training-based methods such as \citet{vtoolr1_2025} and \citet{pixelreasoner_2025} apply
reinforcement learning or supervised fine-tuning on large GPU clusters to teach VLMs to use tools.
Prompt-based methods such as chain-of-thought or few-shot exemplars improve performance on narrow
distributions but do not generalise across benchmark families and require manual curation.

A third approach — letting the agent learn from its own mistakes — has shown promise in text-only
domains~\citep{shinn2023reflexion,wang2023voyager}, but it stops short of what visual reasoning
requires. Reflexion~\citep{shinn2023reflexion} accumulates verbal reflections per case, yet these
reflections are not persistent across cases, never become reusable capabilities, and cannot encode
executable visual processing. No prior work achieves training-free generation of \emph{both}
reusable reasoning strategies \emph{and} executable visual tools that generalise to unseen cases.

We present \sysname{}, a training-free framework in which a VLM agent autonomously evolves two
complementary capability types from failure.
\textbf{Skills} are structured reasoning strategies — standard operating procedures (SOPs)
encoding \emph{when} and \emph{how} to approach a problem class.
\textbf{Tools} are executable Python programs that expand the agent's visual perception by
manipulating images directly.
On each failure, a two-role agent (AnalyzerDecider + Generator) visually diagnoses the root
cause by inspecting both the original image and any intermediate tool outputs, decides whether
the bottleneck is cognitive or perceptual, generates the appropriate capability, validates it,
and accumulates it for future cases — all without updating model weights.

We investigate three facets of training-free evolution through controlled experiments.
\textbf{(I) From-scratch evolution}: starting with zero capabilities, \sysname{} achieves
consistent gains of $+$\textcolor{red}{[TBD]}\%--\textcolor{red}{[TBD]}\% across ChartQA,
MathVista, HRBench, and MMStar~\citep{chen2024mmstar}, with similar relative improvements on
both Gemini-2.0-Flash and Qwen2.5-VL-7B~\citep{qwen25vl}.
\textbf{(II) Tool mastery from provided tools}: when given an external tool set,
naive invocation of all tools yields minimal gain or even degrades accuracy;
mastery learning — where the agent profiles each tool's applicability boundary on a small
training set — recovers and exceeds the full potential of the provided tools.
\textbf{(III) Closing the gap to RL}: training-free skill evolution alone closes approximately
\textcolor{red}{[TBD]}\% of the gap between direct inference and VTool-R1 (RL-trained) on
ChartQA, at a fraction of the cost — suggesting that most of what RL learns on
strategy-bottlenecked benchmarks is capturable as reasoning strategy, not new visual perception.

\paragraph{Contributions.}
\begin{enumerate}[leftmargin=*, label=(\arabic*)]
  \item We propose \sysname{}, a training-free framework for VLM self-evolution from scratch,
        achieving consistent improvements on ChartQA, MathVista, HRBench, and MMStar across
        multiple VLM architectures, with no weight updates.

  \item We show that \textbf{tool mastery learning} is the decisive factor when pre-provided
        tools are available: naive tool invocation provides minimal benefit, while learned
        deployment SOPs recover the tools' full potential.  This distinguishes \sysname{}
        from tool-augmented VLMs that invoke tools indiscriminately.

  \item We show that \textbf{training-free skill evolution closes $\sim$\textcolor{red}{[TBD]}\%
        of the RL performance gap} on strategy-bottlenecked benchmarks, revealing that most of
        what RL training achieves is capturable as structured reasoning strategy without
        any weight updates.

  \item We demonstrate via ablation that \textbf{visual failure analysis} — the AnalyzerDecider
        seeing both the original image and tool artifacts — is necessary for accurate bottleneck
        diagnosis: text-only self-reflection systematically misidentifies perceptual failures
        as strategy failures.
\end{enumerate}
