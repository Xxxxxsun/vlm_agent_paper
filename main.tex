\begin{abstract}
Vision-language models (VLMs) remain brittle on visual reasoning tasks, and improving them
typically requires either retraining or manual prompt/tool design. We ask a different question:
can a frozen VLM agent use its own failed attempts as supervision for acquiring reusable
capabilities? We present \sysname{}, a training-free framework that turns diagnosable failures
into two capability types: \emph{skills}, structured reasoning procedures for task families
where the visual evidence is available but the reasoning procedure is weak, and \emph{tools},
executable Python programs for cases where the agent needs to crop, zoom, enhance, or otherwise
transform the visual input. On each failure, \sysname{} inspects the original image,
intermediate tool outputs, and reasoning trace; decides whether the missing capability is
cognitive, perceptual, or both; generates the appropriate capability; validates it; and
accumulates it for future cases without updating model weights.
Across completed from-scratch evaluations on ChartQA, MathVista, HRBench, and V$^*$,
skill-tool co-evolution improves direct inference in 21 of 23 model-benchmark cells, with an
average gain of 4.8 accuracy points. In a provided-tool setting on GTA, mastery-guided
capability use improves stronger backbones while showing that over-prescriptive tool-use
policies can hurt less compatible models. Against RL-trained VTool-R1, skill evolution alone
recovers 60.0--94.1\% of the RL performance gap on 3B/7B chart and table reasoning splits.
These results suggest that a substantial part of task-specific VLM adaptation can be recovered
by failure-driven, in-context capability evolution.
\end{abstract}
